# Commodity Trading Bot

A full-stack algorithmic trading simulator for Gold and Oil futures. A Python backend fetches live market data, computes technical indicators, and generates trading signals. An Angular dashboard visualises prices, signal history, and a 2-year paper-trading backtest — all in a Bloomberg Terminal-inspired dark UI.

## Getting Started

### Prerequisites

- Python 3.11+
- Node.js 18+ and npm

### 1 — Backend

```bash
cd "ia bot trading"

# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
pip install ta                   # technical-analysis library

# Generate signals and backtest data
python src/fetch_prices.py

# Start the API server (http://localhost:5001)
python src/api.py
```

### 2 — Frontend

```bash
cd trading-dashboard

npm install

# Start the dev server (http://localhost:4200)
ng serve
```

Open [http://localhost:4200](http://localhost:4200) in your browser.

## How It Works

### Signal Logic

Prices are fetched via **yfinance** (90-day window for live display, 2-year window for the backtest). Two indicators are computed with the **ta** library:

- **RSI-14** — Relative Strength Index over 14 periods
- **SMA-20 / SMA-50** — Simple Moving Averages over 20 and 50 periods

Signals are generated using an RSI threshold-crossover strategy:

| Signal | Condition |
|---|---|
| **BUY** | RSI crosses **above 40** (oversold recovery) |
| **SELL** | RSI crosses **below 60** (overbought reversal) |
| **HOLD** | All other rows |

Crossover detection ensures the signal fires only once — on the candle where RSI exits the zone — rather than on every row inside it.

### Paper Trading Engine

`paper_trading.py` simulates a long-only strategy on historical data:

1. **Open** a position (1 unit) on the first BUY signal
2. **Close** the position on the next SELL signal
3. Record entry/exit date, entry/exit price, P&L in $ and %, WIN/LOSS

On each run, `fetch_prices.py` executes two simulations per asset:

- **Backtest (2y)** — fetches 2 years of data to generate a full trade history; overwrites `paper_trades.csv` with a clean slate
- **Live (90d)** — appends any new completed cycles from the most recent 90 days

### Trade Logger

`logger.py` detects rows where the signal changed compared to the previous row (e.g. HOLD → BUY, BUY → SELL) and appends them to `logs/trades.csv`. This gives a compact audit trail of signal transitions rather than every single data point.

## Dashboard Features

**Prices view**
- Asset selector (Gold / Oil)
- Live stats bar: Last Price, RSI(14), Current Signal — with count-up animation on load
- Chart.js line chart: Close price, SMA-20, SMA-50, BUY/SELL markers
- Last 10 rows data table

**Trades view**
- Full signal-change event log
- BUY / SELL / HOLD colour-coded

**Paper Trading view**
- Summary card: Total Trades, Win Rate, Total P&L, Current Capital
- Full closed-trades table with WIN rows highlighted in green, LOSS rows in red

## Notes

- Flask runs on port **5001** (macOS Sequoia reserves port 5000 for AirPlay Receiver)
- The backtest resets `paper_trades.csv` on every `fetch_prices.py` run to avoid accumulating duplicates
- No real capital is ever used — this is a simulation only
